#### Класс Api

__Поле baseUrl__
Нужно для сохранние базовой части URL, чтобы не дублировать URL при кажорм новом запросе (Пример: для работы с продуктами ссылаемся на URL с продуктами, для работы с заказом на URL с закзами ).
Берется основной URL(базовый), который не изменчив(по отношению к элементам из backend), а get и post запрос указываем (в конце) для той или иной задачи в рамках проекта.

__Поле options__
Это объект настроек запроса в частности поле method со значением типа запроса (get, post, delete), headers и поле body в которе попдает объект заказа, который мы должны отправить на сервер

Конструктор (приравнивается к одноименным ключам, чтобы делать фетч запросы)

Конструктор класса, который принимает два параметра: baseUrl типа string и options типа RequestInit (по умолчанию пустой объект).

Внутри конструктора происходит инициализация свойств класса: baseUrl устанавливается равным переданному значению, а options формируется объект, который содержит headers с ключом 'Content-Type' и значением 'application/json'.

Если в переданных options есть свойство headers, то оно распыляется (spread) в новый объект, чтобы сохранить другие заголовки, если они были переданы.
Таким образом, конструктор устанавливает базовый URL и опции для будущих запросов в классе.

Защищенный метод ___handleResponse___, который принимает Response и возвращает Promise, который разрешается в объект.
Внутри метода проверяется свойство ok объекта Response (в связи с булевым правилом от if). Если ответ от сервера успешный (response.ok равен true), то возвращается результат вызова метода .json() для получения данных в формате JSON.
Если ответ от сервера неуспешный(false), то также вызывается метод .json() для получения данных в формате JSON. Затем происходит обработка данных: если в полученных данных есть свойство error, то создается исключение с текстом ошибки из данных, иначе используется текст статуса ответа (response.statusText).

Таким образом, метод handleResponse обрабатывает ответ от сервера, возвращая либо данные в формате JSON в случае успешного ответа, либо создавая исключение с текстом ошибки в случае неуспешного ответа.

___get___ с одним параметром uri типа string. Внутри функции происходит вызов fetch(с помощью данной функции можно отправлять сетевые запросы на сервер а в частности как получать, так и отправлять данные) для отправки GET запроса на указанный uri, используя базовый URL и опции, переданные в объекте. После этого выполняется метод .then для обработки полученного ответа с помощью функции handleResponse

___post___ с тремя параметрами: uri типа string, data типа object и method типа ApiPostMethods по умолчанию равный 'POST'. 
Внутри функции происходит вызов fetch для отправки POST запроса на указанный uri, используя базовый URL и опции, переданные в объекте. 
Также указывается метод запроса (POST или другой, если указан явно) и данные в формате JSON, переданные в теле запроса. 
Затем выполняется метод .then для обработки полученного ответа с помощью функции handleResponse.

#### Класс LocalStorage 
Класс реализующий хранение какие-либо пользовательских настройк клиента, служебную информацию, различные изменения состояния в процессе взаимодействия со страницей, например, данные заполненных полей формы, чтобы они не удалялись после обновления веб-страницы.
В данном проекте это реализуется в модальных окнах включающие в себя способе оплаты(онлайн или при получении ), инпуты(адрес доставки, email, телефон)



 класс ___LocalStorage___ с двумя методами: get и set.
>1. Метод get(key: string): 
   - Получает значение из локального хранилища по указанному ключу.
   - Проверяет, есть ли данные с указанным ключом в локальном хранилище.
   - Если данные есть, то преобразует их из JSON в объект и возвращает результат.
   
>2. Метод set Value(key: string, value: Value): 
   - Принимает ключ и значение, которое нужно сохранить в локальном хранилище.
   - Преобразует значение в формат JSON.
   - Устанавливает значение в локальное хранилище по указанному ключу.

Таким образом, класс LocalStorage предоставляет функциональность для работы с локальным хранилищем браузера, позволяя получать и устанавливать данные по ключу.

 Класс Validation, содержит методы для валидации формы на веб-странице:
- Конструктор класса принимает объект Order и сохраняет его в свойстве order.

   - Проверяет, есть ли невалидные данные в переданном списке HTMLInputElement.

Таким образом, класс Validation предоставляет функциональность для валидации форм на веб-странице и управления их состоянием.

#### Класс Basket (работа с данными массива корзины)

Поле basketItems (товары которые были добавлены в массив корзины )
Поле selectedProduct (направленое добавление товара в корзину, происходит сохрание из backend объекта (push объекта в массив))

Осуществялется работа по:
-добавлению данных в корзину,за счет функции addBasket()
-удаление данных из корзины, за счет функции removeBasket()
-установка счетчика в лого-html о колличестве товаров в массиве корзины за счет функции  showBasketCounter()
-подсчет общей суммы за все товары положеные в массив корзины за счет функции calculateTotalPrice()
-очистка всего массиыва корзины по окончания покупки(офрмления заказа) за счет функции cleanBasket()

Конструктор включает в себя работу с двумя параметрами (modal типа Allmodal, basket типа Basket)

 Класс Basket, который отвечает за управление товарами в корзине на веб-странице:
- Свойства класса:
   - basketItems: массив объектов Product, представляющий товары в корзине.
   - selectedProduct: текущий выбранный товар для добавления в корзину.
   - storage: объект LocalStorage для сохранения данных о товарах в локальном хранилище.

- Конструктор класса:
   - Принимает объект LocalStorage и инициализирует пустой массив basketItems.

- Метод addBasket():
   - Добавляет выбранный товар в корзину, если его еще нет в списке товаров.
   - Сохраняет обновленный список товаров в локальное хранилище.

- Метод removeBasket(productID: string):
   - Удаляет товар из корзины по его идентификатору.
   - Обновляет список товаров в локальном хранилище.

- Метод showBasketCounter():
   - Возвращает количество товаров в корзине.

- Метод calculateTotalPrice():
   - Вычисляет общую стоимость всех товаров в корзине.

- Метод cleanBasket():
   - Очищает корзину, удаляя все товары из массива.

Класс Basket предоставляет функциональность для управления товарами в корзине, включая добавление, удаление, отображение количества товаров и подсчет общей стоимости.


#### Класс Order осуществляет работу с заказаом товара(ов)

Класс Order состоит из следующих частей:
___Свойства класса___
- payment: строковое свойство, предназначенное для хранения информации об оплате заказа.
- address: строковое свойство, предназначенное для хранения адреса доставки заказа.
- email: строковое свойство, предназначенное для хранения email-адреса клиента.
- phone: строковое свойство, предназначенное для хранения номера телефона клиента.

___Конструктор класса___

Конструктор класса принимает один параметр __basket__ типа Basket и инициализирует свойство basket класса Order.
В конструкторе также инициализируются свойства payment, address, email и phone пустыми строками.

___Методы класса___

- __setParam(key: string, value: string)__
   - Метод setParam принимает два параметра: key типа string и value типа string.
   - Внутри метода происходит динамическое присваивание значения value свойству класса, соответствующему ключу key.
    - Используется синтаксис this[key] = value для динамического доступа к свойствам класса.

- __returnOrder()__

   - Метод returnOrder возвращает объект, содержащий информацию о заказе.
   - Объект содержит следующие свойства:
      - payment: значение свойства payment класса Order.
      - address: значение свойства address класса Order.
      - email: значение свойства email класса Order.
      - phone: значение свойства phone класса Order.
      - items: массив идентификаторов продуктов в корзине, полученный с помощью метода map из свойства basketItems объекта basket.
      - total: результат вызова метода calculateTotalPrice() объекта basket, возвращающий общую стоимость заказа.

В целом, данный код определяет класс Order, который хранит информацию о заказе, включая корзину с товарами, способ оплаты, адрес доставки, email и номер телефона клиента. Класс предоставляет методы для установки параметров заказа и получения объекта с информацией о заказе.

#### Класс ContactsUI  форма, где пользователь должен ввести свои контакты для заказа

класс ContactsUI, отвечает за взаимодействие с пользовательским интерфейсом для ввода контактных данных в модальном окне.

Класс состоит из следующих частей:
   - Свойство contactsInputs: Это NodeListOf<HTMLInputElement>, которое будет хранить список всех полей ввода контактных данных в модальном окне.
   - Конструктор класса: Он принимает три параметра:
      - modal: Экземпляр класса AllModals, который отвечает за управление модальными окнами.
      - order: Экземпляр класса Order, который представляет заказ.
      - onSubmit: Функция обратного вызова, которая будет вызываться при отправке формы.
      
- Инициализация contactsInputs: В конструкторе происходит инициализация свойства contactsInputs. Оно получает список всех полей ввода контактных данных в модальном окне с помощью querySelectorAll('.form__input').
 - Обработчики событий для полей ввода: Для каждого поля ввода в contactsInputs устанавливается обработчик события input. При вводе данных в поле вызывается анонимная функция, которая:

      - Получает ссылку на элемент, вызвавший событие, с помощью event.target as HTMLInputElement.
      - Вызывает метод setParam класса Order, передавая в качестве параметров имя поля (target.name) и введенное значение (target.value).
- Обработчик события submit для формы: Для модального окна с контактными данными устанавливается обработчик события submit. При отправке формы вызывается анонимная функция, которая:

- Вызывает метод preventDefault() для предотвращения стандартного поведения формы.
- Вызывает функцию обратного вызова onSubmit, переданную в конструктор класса.

 Происходит связывание полей ввода контактных данных в модальном окне с объектом заказа (Order) и обработку события отправки формы, вызывая переданную функцию обратного вызова.


#### Класс AllModals

Класс AllModals, который расширяет класс Render. Он отвечает за управление модальными окнами в приложении.

Класс состоит из следующих частей:

- Свойства класса:
      - modal: Элемент DOM, представляющий контейнер для всех модальных окон.
      - modalContent: Элемент DOM, представляющий содержимое модального окна.
      - modals: Список всех модальных окон в приложении.
      - cardModal: Элемент DOM, представляющий модальное окно для просмотра карточки товара.
      - basketModal: Элемент DOM, представляющий модальное окно для корзины.
      - orderModal: Элемент DOM, представляющий модальное окно для оформления заказа.
      - contactsModal: Элемент DOM, представляющий модальное окно для ввода контактных данных.
      - successModal: Элемент DOM, представляющий модальное окно для отображения успешного оформления заказа.

- Конструктор класса:

      - Вызывает конструктор родительского класса Render.
      - Инициализирует свойства класса, получая соответствующие элементы DOM с помощью querySelector и querySelectorAll.

- Метод setModal:

      - Принимает объект Product в качестве параметра.
      - Обновляет содержимое модального окна для просмотра карточки товара (cardModal) с использованием данных из объекта Product.
      - Устанавливает текст и классы для категории товара, названия, описания, цены и кнопки.
      - Блокирует кнопку, если цена товара не установлена.

- Метод openModal:
      - Принимает элемент модального окна в качестве параметра.
      - Очищает содержимое modalContent.
      - Добавляет переданный элемент модального окна в modalContent.
      - Добавляет класс modal_active к контейнеру модального окна, чтобы сделать его видимым.

- Метод setCloseListeners:
      - Устанавливает обработчики событий click для закрытия модального окна.
      - При клике на контейнер модального окна или кнопку закрытия удаляет класс modal_active, скрывая модальное окно.

Данный класс отвечает за управление модальными окнами в приложении, включая инициализацию элементов модальных окон, обновление их содержимого, открытие и закрытие модальных окон.

Принимает массив элементов array, функцию element для создания HTML элемента на основе каждого элемента массива, контейнер container, и опциональную функцию eventListener для добавления обработчика событий.
